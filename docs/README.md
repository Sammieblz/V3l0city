# V3l0city Documentation

Start here if you are deciding which guide to read.

## For Developers

- [Developer guide](developer-guide.md): short entry point and recommended
  reading order.
- [Developer docs directory](developer/README.md): technical docs split by
  subsystem.
- [System architecture](developer/system-architecture.md): runtime boundaries,
  data flow, local-first rules, and product constraints.
- [Mobile frontend](developer/mobile-frontend.md): app boot, dashboard,
  settings, trip lifecycle, and UI state.
- [Native speed engine](developer/native-speed-engine.md): Expo Module bridge,
  platform collectors, C++ core, and diagnostics.
- [Local data and export](developer/local-data-and-export.md): SQLite schema,
  repositories, migrations, trip samples, and export behavior.
- [Telemetry and backend](developer/telemetry-and-backend.md): mobile telemetry
  orchestration, Fastify server, auth, WebSocket, and retry model.
- [Cloud sync and social](developer/cloud-sync-and-social.md): optional
  Supabase auth, local-wins sync, friends, nearby discovery, and leaderboards.
- [Testing and operations](developer/testing-and-operations.md): commands,
  simulator flows, environment variables, and troubleshooting.
- [Widgets and car surfaces](developer/widgets-and-car-surfaces.md): live drive
  sessions, iOS Live Activity/widget behavior, Android widget variants, and car
  surface constraints.
- [Code ownership map](developer/code-ownership.md): where to make changes and
  what to validate.
- [Native speed engine](speed-engine.md): deeper details on iOS/Android sensor
  collection, C++ speed rules, quality diagnostics, tuning constants, and
  simulator validation.
- [Telemetry API](telemetry-api.md): backend setup, HTTP contracts, WebSocket
  contracts, database tables, authentication, and retry semantics.

## For Users and Support

- [User guide](user-guide.md): non-technical instructions for using V3l0city on
